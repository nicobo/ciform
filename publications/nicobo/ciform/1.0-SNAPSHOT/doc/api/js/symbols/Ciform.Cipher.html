Class Ciform.Cipher

This class provides a simple way to encrypt different kind of elements (text, form fields, URL).
It has first to be built with options to define the encryption ; then one of its method must be called on the element to encrypt.
Defined in: ciform.js.

{boolean|Ciform.Field} encryptField(target, options)
Encrypts an HTML field.
Parameters:
{HTMLElement} target
The element to encrypt.
As an alternative, this parameter can be ignored if the element is given in the options.
{Object} options
(optional) An object with the following properties : As a convenience, this may be the field itself, all default options will then be applied.
  • <Ciform.Field> field - Same as (replaces) target.
  • <boolean> commit - If true, will write ciphertext into the output field.
    If false, will not touch the in and out field, but instead will return the Ciform.Field with the following properties added :
    • <String> text - The value of the input field at the end of the operation.
    • <String> ciphertext - The value of the output field at the end of the operation.
    Defaults to true.
    This parameter is usefull to handle transactional operations on several fields at once.
As an alternative,sha1 or ciform-sha1 css classes can be set in the class attribute of output fields, in order to set the Ciform.Field#sha1 property to 1 or 2 respectively.
Returns:
false if there was an error, true or a Ciform.Field object if not

{boolean} encryptForm(target, options)
Encrypts fields of a form.

For instance, the following encrypts input fields of type password in a form : 

encryptForm(myForm,{'allowTypes':["password"]})

Parameters:
{HTMLFormElement} target
The form to encrypt.
As an alternative, this parameter can be ignored if the target is given in the options.
{Object} options
(optional) An object with the following properties :
  • <HTMLFormElement> form - Same as (replaces) target argument.
  • <Array> allowTypes - If set, this array lists the only 'type' attributes allowed for an <INPUT> tag to be encrypted. If not set, all tags are allowed.
  • <Array> rejectTypes - If set, this array lists the 'type' attributes of the <INPUT> tags not to encrypt. If not set, all tags are allowed.
Returns:
false if there was an error, true if not

{String} encryptURL(url, options)
Encrypts the parameters in an URL.
Parameters:
{String} url
The full URL to encrypt (e.g. "http://plugnauth.sf.net/ciform/demo.php?password=mysecret")
{Object} options
(optional) An object with the following properties :
  • <Array<String>> fields - The list containing the name of the fields that must be encrypted (if not present, all detected fields are encrypted)
Returns:
The same URL, with the fields encrypted.
E.g. "http://plugnauth.sf.net/ciform/demo.php?password=mysecret" could become "http://plugnauth.sf.net/ciform/demo.php?password=ciform:rsa:0x2137230230234832423723"
